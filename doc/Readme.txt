1、蜂鸣器：（beep_driver.c）
	PC9引脚；	无源蜂鸣器；	
	调试：先初始化引脚，然后给出高低电平控制；
		  给出“BEEP_Contrl(0);”直接关闭；给出间隔400-500us的高低电平切换可以开蜂鸣器。
		  （用pwm加震荡源的方法可以是蜂鸣器鸣叫，但是声音不对。）

2、红外模块：（参考代码：infrared.c）
	引脚是PB5
	初始化GPIO口，然后检测out引脚（PB5）的高低电平就好

3、调试串口1&3：（参考代码：usart_driver.c）
	usart1：大致的步骤可以参考老师讲的方法以及上课写的demo，正确的作出一些小的修改之后可以拿到自己的工程下使用就行。
			其过程也是
			①、看原理图找出对应的电路连接（USART1_Tx-->GPIOA9 & USART1_Rx-->GPIOA10）.
			②、初始化GPIO引脚，usart mode config（usart模式配置），中断（NVIC config）； 
			③、然后到中断处理文件（stm32f10x_it.c）写好中断函数；
			④、最后到主函数使能就好了。
			
	usart3重映射：（硬件有很多不支持，要多次测试以及用多块板子测试）
			①、引脚：PC10（TX），PC11（RX）；
			②、在usart1的基础上加上 AFIO的时钟初始化
			③、添加重映射（GPIO_PinRemapConfig(GPIO_PartialRemap_USART3, ENABLE);//usart3复用功能完全映射）
			/*注意：只能使用部分重映射，不能使用全部重映射*/
			④、初始化GPIO引脚
			⑤、中断处理函数可以参考usart1
4、GSM/GPRS模块：（gprs_driver.c）
			①、在调试好usart3的基础上开始调试；
			②、先测试模块的好坏，参考果云调试助手（A6_TEST_TOOLS.exe）
				a、接好模块，连接好电源，GND（连接了电源之后还要连接地是需要共地，保证模块的正常运行），、还有u-tx，u-rx和USB转串口对应的引脚连接；
				b、按住模块的按键大概两到三秒，led指示灯开始闪烁方可松开。正确的现象是闪烁之后指示灯常亮。
				c、插上电话卡，用调试助手来测试相对应的功能，每一次命令显示OK就是代表功能正常。
				常用命令：
					①、
					
			③、在usart和模块都已经调好的状态下去移植官方给出的代码（详情见项目资料）
			④、从打电话的功能开始，逐个添加，最好是添加到自己的工程，像其他模块一样单独创建文件夹和对应的c和h文件。

5、定时器：（12.8任务）
	以超声波模块计时为例：
	①、定时器处理方式比较简单，配置定时器，设置模式，周期，预分频器等。
	②、设置某种触发方式，写while语句或者中断触发。
	③、过程大概是，清零计时器，打开定时器开始计时，达到某一条件就停止计时。最后再去读取就好了。


6、超声波模块：（12.8任务）
	引脚是PC10（TRIG）、PC11（ECHO）
	初始化GPIO口，给TRIG一个10us的高电平触发
	再检测ECHO引脚的高电平持续时间
	根据公式	测试距离=(高电平时间*声速(340M/S))/2  来计算测试距离
